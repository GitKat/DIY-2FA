SMS Verification

SMS Verification is used for second factor authentication,
& to verify ownership of a phone number or device.

Browse Verification Example

Verification Service

The example code is a simple PHP script implementing a verification service on top of Nexmo's API.

  • A number is traded for a hash.
  • The verification service sends an SMS with a PIN to that number.
  • That PIN is provided by the end user.
  • Number, hash, and PIN are passed to the verification service to get the result.

Example Code?

If you're a developer, the provided example should be easy to take and modify to your needs. If you're not, you can find a developer with Nexmo experience on oDesk, and point them to this page.

Second Factor Authentication

Adding a second factor to a login system, in this case a mobile phone, increases security as a physical device is needed in addition to the username and password.

For a web application, after validating a credential pair the application would request additional verification from a service similar to this example.

Number Ownership

Verifying that a user is the owner of a phone number is a common mobile application use case. On signup or install, the mobile application would collect the phone number. Then the application would use a service similar to this example to ensure the user owns the phone number.
